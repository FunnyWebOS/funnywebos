Bienvenue sur AetherOS

AetherOS (FunnyWEB OS) est un système d'exploitation web moderne, fluide et entièrement personnalisable. Conçu avec une architecture "Blue Glass", il offre une expérience utilisateur premium directement dans le navigateur.

v3.0 : Singularity

La version 3.0 standardise l’UI (dock, fenêtres, réglages) et ajoute des préférences utilisateur (dont les notifications) pour une expérience plus professionnelle.

À quoi sert ce Wiki ?

Ce wiki est destiné aux développeurs et aux utilisateurs curieux qui souhaitent comprendre comment fonctionne l'OS ou comment créer leurs propres expériences au sein de l'écosystème FunnyWEB!.

Créer une Application Premium

Standard v3

AetherOS permet de créer des applications "ultra-fines" qui s'intègrent parfaitement à l'OS.

1. Ajouter votre app

  • Créez un fichier .html dans le dossier /apps/ (ex: meteo.html).
  • En local : votre app peut apparaître automatiquement dans l'OS.
  • Sur GitHub Pages : ajoutez une entrée dans apps/registry.json (pas de listing de répertoire).

1.1 Exemple registry.json

{
  "id": "meteo",
  "title": "Météo",
  "creator": "Moi",
  "description": "Ma première app.",
  "category": "productivity",
  "icon": "☀️",
  "appFile": "meteo.html"
}

2. Template de base

<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <style>
        :root {
            --accent: #0A84FF;
            --glass: rgba(10, 25, 45, 0.45);
        }
        body {
            margin: 0; padding: 40px;
            font-family: -apple-system, system-ui, sans-serif;
            background: transparent; color: white;
            height: 100vh; box-sizing: border-box;
            display: flex; flex-direction: column;
        }
        .main-app {
            flex: 1;
            background: var(--glass);
            backdrop-filter: blur(30px);
            border: 1px solid rgba(255,255,255,0.15);
            border-radius: 16px;
            padding: 40px;
            text-align: center;
        }
    </style>
</head>
<body>
    <div class="main-app">
        <h1>Mon App FunnyWEB!</h1>
        <p>Prêt pour le futur.</p>
    </div>
</body>
</html>

Design System : Blue Glass

Toutes les applications doivent respecter le style Bleu Vitreux pour une immersion totale.

Couleurs et Tokens

:root {
    --accent: #0A84FF;           /* Bleu Action */
    --glass: rgba(10, 25, 45, 0.45); /* Bleu Glass Pro */
    --border: rgba(255, 255, 255, 0.15); /* Bordure subtile */
    --text: #ffffff;
    --text-dim: rgba(255, 255, 255, 0.6);
}

Effet Glassmorphism Premium

Le fond du body doit être transparent pour que le flou de la fenêtre agisse.

body {
    background: transparent;
    color: var(--text);
    margin: 0;
}

.native-card {
    background: var(--glass);
    backdrop-filter: blur(25px) saturate(180%);
    border: 1px solid var(--border);
    border-radius: 12px;
}

API Utilisateur (Synchronization)

AetherOS transmet l'identité de l'utilisateur à l'ouverture de chaque application.

window.addEventListener('message', (event) => {
    if (event.data.type === 'funnyweb_user_sync') {
        const { userName, sessionID, profilePic, theme, uiPreferences } = event.data;
        // Utilisez ces données pour personnaliser l'expérience
    }
});

Données disponibles

  • userName: Nom affiché de l'utilisateur.
  • sessionID: ID unique de la session actuelle.
  • profilePic: Image de profil (Base64 ou Emoji).
  • theme: Thème actuel (light/dark).
  • uiPreferences: Préférences UI (dock, notifications, etc.).

Notifications

Le système peut afficher des notifications (toasts). Les utilisateurs peuvent tout désactiver ou filtrer par type dans Paramètres > Notifications.

Envoyer une notification depuis une app

window.parent?.windowManager?.notify(
  'Mon App',
  'Hello depuis mon app',
  'system'
);

Types (exemples)

  • system: alertes générales
  • settings_change: changements de paramètres
  • accounts: connexion/déconnexion cloud
  • install: installation/suppression d’apps
  • file: opérations fichiers/VFS

Virtual File System (VFS)

Vous pouvez lire et écrire des fichiers dans le stockage virtuel de l'utilisateur.

Écrire un fichier

window.parent.postMessage({
    type: 'OS_VFS_WRITE',
    path: '/Documents/mon_fichier.txt',
    content: 'Contenu du fichier',
    nodeType: 'file'
}, '*');

Écouter les mises à jour

window.addEventListener('message', (e) => {
    if (e.data.type === 'funnyweb_vfs_update') {
        const vfs = e.data.vfs;
        // Mettre à jour l'interface avec le nouveau VFS
    }
});

Règles d'or & Bonnes Pratiques

1. Pensez Transparent

Votre body doit TOUJOURS être background: transparent. C'est ce qui permet de voir le fond d'écran de l'OS derrière vos éléments vitreux.

2. Responsivité (Stretching)

Votre application doit remplir la fenêtre. Utilisez height: 100vh et width: 100%. Utilisez Flexbox ou Grid pour organiser votre contenu.

3. Cohésion Visuelle

Utilisez le bleu accent #0A84FF pour les actions principales. Évitez les couleurs trop flashies qui cassent l'immersion Blue Glass.
